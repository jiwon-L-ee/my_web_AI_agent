# Plan: 맞불 플랫폼 버그 수정 판정

## 1. 개요

맞불 플랫폼(platform/)에 대한 CTO 팀 종합 분석 결과 도출된 버그를 수정하고,
수정 전후 상태를 Gap 분석으로 검증한다.

## 2. 분석 결과 요약

- **종합 점수**: 78/100
- **코드 품질**: 20/25
- **UI/UX**: 21/25
- **기능 완성도**: 19/25
- **보안**: 18/25

## 3. 수정 목표

### 긴급 (즉시 수정 완료)
1. `post.js` deletePost() — `.eq('user_id', currentUser.id)` 추가
2. `mypage.js` deletePost() — `.eq('user_id', currentUser.id)` 추가
3. `create.html` — `<option value="정보">정보</option>` 추가

### 중요 (즉시 수정 완료)
4. `post.js` sideTag — `escapeHtml()` 이중 인코딩 제거 (textContent에 raw 값 사용)

## 4. 검증 기준

- 게시물 삭제 시 user_id 조건 포함 여부
- 정보 카테고리 작성 가능 여부
- sideTag 선택지명 정상 표시 여부
- 전체 코딩 표준 준수율 90% 이상

## 5. 파일 목록

| 파일 | 변경 내용 |
|------|-----------|
| `platform/js/post.js` | deletePost user_id 조건 추가, sideTag 이중인코딩 수정 |
| `platform/js/mypage.js` | deletePost user_id 조건 추가 |
| `platform/create.html` | 정보 카테고리 option 추가 |
