# Plan: 플랫폼-UX-전면개선

## 배경 (계획서.txt 기반)

사용자 계획서에서 도출된 6가지 개선 항목:

1. **create 쪽 문제** (매우중요) — UI/UX 불편함, 커뮤니티는 제목+내용 심플 폼이어야 함
2. **회원탈퇴 문제** — mypage에 탈퇴 기능 없음
3. **로그인/회원가입 분할** — 탭 구조는 있으나 명확성 부족
4. **이용약관/개인정보처리방침 텍스트 이상함** — 법적 문서 내용 점검
5. **푸터 항목들 클릭시 이동 안함** — footer-link-todo 비활성 상태
6. **고객센터 활성화 + 다크/라이트모드**

디자인 컨셉: "불" 일관 유지

---

## 범위 (구현 대상)

### P0 — 매우중요 (즉시 적용)

#### 1. create 페이지 UX 전면 개선
- 카테고리 선택을 탭 버튼 방식으로 교체 (현재 select → 아이콘+레이블 탭)
- 커뮤니티/정보 카테고리: 제목 + 내용 텍스트에리어만 표시 (그리드 레이아웃 제거)
- 밸런스게임: 제목 + A/B 선택지 + 설명
- 퀴즈: 제목 + 퀴즈빌더
- 테스트: 제목 + 모델URL + 설명
- 단계별 힌트 텍스트 추가 (각 필드가 무엇인지 설명)
- 모바일에서도 직관적으로 사용 가능하게

#### 2. 회원탈퇴 기능
- mypage.html 하단에 "계정 설정" 섹션 추가
- "회원탈퇴" 버튼 → 확인 모달 (2단계 확인)
- Supabase auth.admin.deleteUser() or auth.signOut() + 데이터 처리
- 탈퇴 시 posts는 익명 처리 또는 삭제 (UX 결정 필요)

### P1 — 중요 (이번 배포)

#### 3. 이용약관/개인정보처리방침 활성화
- 푸터의 `footer-link-todo` → 실제 링크로 교체
- 이용약관/개인정보처리방침: 클릭 시 legal 모달 표시 (login.html의 모달 패턴 재사용)
- 각 페이지에 법적 모달 HTML + JS 추가 (또는 공통 모달 스크립트)
- 커뮤니티 가이드라인: 별도 텍스트 파일 생성 후 모달로 표시

#### 4. 로그인/회원가입 탭 명확화
- 탭 3개 → 로그인/회원가입 2개로 단순화 (Google은 각 탭 내부)
- 또는 URL 파라미터 지원: `login.html?tab=signup`

#### 5. 고객센터 (문의하기) 활성화
- 간단한 문의 안내 페이지 또는 모달 (이메일 링크)
- 푸터 "문의하기" → `mailto:` 또는 contact 모달

### P2 — 개선 (이번 배포)

#### 6. 다크/라이트모드 토글
- 네비게이션 바 우측에 아이콘 버튼 추가
- CSS: `body.light-mode { --bg: #f8f8f8; --surface: #fff; ... }` 라이트 변수 재정의
- localStorage에 사용자 선택 저장
- "불" 컨셉: 라이트모드에서도 accent 색상(주황/레드) 유지
