### [SEC-01] settle-balance-games Edge Function 인증 추가

**파일**: `supabase/functions/settle-balance-games/index.ts`

**Before**: CORS `*`, 인증 없음

**After**:
```typescript
const CORS = {
  'Access-Control-Allow-Origin': 'https://matbul.com',
  'Access-Control-Allow-Headers': 'authorization, x-client-info, apikey, content-type',
};

Deno.serve(async (req) => {
  if (req.method === 'OPTIONS') {
    return new Response('ok', { headers: CORS });
  }

  // JWT 검증
  const authHeader = req.headers.get('Authorization');
  if (!authHeader?.startsWith('Bearer ')) {
    return new Response(JSON.stringify({ error: 'Unauthorized' }), {
      status: 401, headers: { ...CORS, 'Content-Type': 'application/json' }
    });
  }

  // 사용자 검증 (anon key로 auth 클라이언트 생성)
  const authClient = createClient(
    Deno.env.get('SUPABASE_URL')!,
    Deno.env.get('SUPABASE_ANON_KEY')!,
    { global: { headers: { Authorization: authHeader } } }
  );
  const { data: { user }, error: authErr } = await authClient.auth.getUser();
  if (authErr || !user) {
    return new Response(JSON.stringify({ error: 'Unauthorized' }), {
      status: 401, headers: { ...CORS, 'Content-Type': 'application/json' }
    });
  }

  // 기존 supabase (SERVICE_ROLE) 클라이언트로 정산 로직 계속
  const supabase = createClient(
    Deno.env.get('SUPABASE_URL')!,
    Deno.env.get('SUPABASE_SERVICE_ROLE_KEY')!
  );
  // ... 이하 기존 코드 동일
```

**호출부 수정** (`platform/js/post.js` 내 `settle-balance-games` 호출 위치):
```js
// 세션 토큰 포함하여 호출
const { data: { session } } = await db.auth.getSession();
await fetch(`${SUPABASE_URL}/functions/v1/settle-balance-games`, {

    'Content-Type': 'application/json',

    'apikey': SUPABASE_ANON_KEY,
  },
  body: JSON.stringify({ post_id: postId }),
});
```

### [BUG-06] loadMyComment 다중 행 오류

**파일**: `platform/js/post.js:877-884`

**Before**:
```js
async function loadMyComment() {
  if (!currentUser) return;
  const { data } = await db.from('comments')
    .select('id,content,side')
    .eq('post_id', postId)
    .eq('user_id', currentUser.id)
    .maybeSingle();  // ← 루트+리버튼 시 다중 행 → 오류
  myComment = data ?? null;
}
```

**After**:
```js
async function loadMyComment() {
  if (!currentUser) return;
  const { data } = await db.from('comments')
    .select('id,content,side')
    .eq('post_id', postId)
    .eq('user_id', currentUser.id)
    .is('parent_id', null)   // ← 루트 댓글만 조회
    .maybeSingle();
  myComment = data ?? null;
}
```

### [BUG-03/04] URL.createObjectURL 메모리 누수

// 파일 상단 상태 변수에 추가
let communityObjectUrls = [];   // 커뮤니티 이미지 object URL 추적
let questionObjectUrls = [];    // 퀴즈 문항 이미지 object URL 추적

function renderCommunityPreview() {
  const preview = document.getElementById('communityImgPreview');
  if (!preview) return;
  // 이전 URL 해제
  communityObjectUrls.forEach(u => URL.revokeObjectURL(u));
  communityObjectUrls = [];

  preview.innerHTML = communityImages.map((f, i) => {
    const url = URL.createObjectURL(f);
    communityObjectUrls.push(url);
    return `<div class="img-preview-item">
      <img src="${escapeHtml(url)}" alt="">
      <button type="button" class="img-preview-remove" data-idx="${i}" aria-label="삭제">×</button>
    </div>`;
  }).join('');
}

// buildQuestionCardHTML 내 이미지 미리보기도 동일 패턴 적용
// renderQuestionList() 호출 전: questionObjectUrls.forEach(u => URL.revokeObjectURL(u)); questionObjectUrls = [];
```

function tickUrgent() {

  let hasUrgent = false;
  urgentEls.forEach(el => {
    const remaining = parseDateMs(el.dataset.expires) - Date.now();
    if (remaining > 0 && remaining < 24 * 3600 * 1000) hasUrgent = true;
    updateCountdown(el, remaining);

  if (!hasUrgent && urgentIntervalId) {
    clearInterval(urgentIntervalId);
    urgentIntervalId = null;
    normalIntervalId = setInterval(tickNormal, 60_000);
  }
}
```

## 4. 회귀 테스트 포인트

| 기능 | 테스트 항목 |
|------|------------|
| 밸런스게임 | 루트 댓글 + 리버튼 작성 후 post.html 재접속 → myComment 정상 로드 확인 |
| 리버튼 | 크레딧 0 상태에서 리버튼 → 오류 메시지 + 댓글 미생성 확인 |
| 정산 | 로그인 상태에서 만료 게시물 settle 호출 → 정상 정산 확인 |
| 정산 | 비로그인 상태에서 settle API 직접 호출 → 401 반환 확인 |
| 커뮤니티 검색 | `%`, `_` 입력 → 의도하지 않은 전체 노출 없는지 확인 |
| 팔로우 | 팔로워 1000+ 계정 팔로우/언팔로우 → 카운트 정확성 확인 |
| 이미지 업로드 | create.html에서 이미지 반복 추가/삭제 → 메모리 증가 없는지 확인 |
