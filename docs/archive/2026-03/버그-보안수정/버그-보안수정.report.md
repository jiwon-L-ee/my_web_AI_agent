## 6. 주요 수정 내용 분석

### 6.1 보안 강화

#### SEC-01: Edge Function JWT 인증 추가 (가장 Critical)
- **문제**: `settle-balance-games` 함수가 인증 없이 호출 가능 → 누구나 다른 사용자의 크레딧을 무단으로 정산 가능
- **해결**: Bearer 토큰 검증 로직 추가 + `Authorization` 헤더 필수화
- **영향**: 크레딧 경제 무결성 보호, 중요 권한 작업의 인증 강제

#### SEC-02, SEC-03: SQL & CORS 제한
- `LIKE` 쿼리의 와일드카드 문자(`%`, `_`) 이스케이프로 SQL 인젝션 위험 제거
- Edge Function 2개의 CORS를 `*` 에서 `https://matbul.com` 으로 제한

#### SEC-04, SEC-10: Input Validation
- `escapeHtml()` 함수에 작은따옴표 이스케이프 추가 (HTML 속성값 공격 방어)
- 프로필 페이지의 UUID 파라미터 정규식 검증 추가

### 6.2 기능 안정성 개선

#### BUG-06: loadMyComment 오류 (즉시 운영 영향)
- **문제**: 루트 댓글과 리버튼(대댓글)을 동시에 작성한 사용자가 post.html 재접속 시 `.maybeSingle()` 에러
- **해결**: `.is('parent_id', null)` 필터로 루트 댓글만 조회
- **테스트**: 리버튼 작성자의 마이 댓글 로드 정상화 확인

#### BUG-05: 리버튼 크레딧 원자성
- **문제**: 댓글 삽입 후 크레딧 차감 → 크레딧 차감 실패 시 댓글만 남음
- **해결**: 순서 변경 — 크레딧 먼저 차감 후 성공 시만 댓글 삽입
- **제약**: 현재 롤백 미지원이므로 차감 실패 시 DB 상태 일관성 보장

#### BUG-08, BUG-09: 성능 및 리소스 누수
- `Promise.all()` 로 프로필/마이페이지의 직렬 await 병렬화 → 페이지 로드 속도 개선
- `clearTimeout()` 누락으로 인한 타이머 누수 제거

### 6.3 메모리 최적화

#### BUG-03/04: Object URL 누수
- **문제**: 이미지 업로드 시마다 `URL.createObjectURL()` 생성 후 revoke 없음 → 메모리 누적
- **해결**: 배열에 Object URL 추적 후 재렌더 시 `revokeObjectURL()` 호출
- **효과**: 사용자가 반복적으로 이미지를 업로드/삭제할 때 메모리 누수 방지

## 7. Lessons Learned & Retrospective

### 7.1 What Went Well (Keep)

1. **체계적인 보안 분석**: security-architect + code-analyzer 에이전트의 조합으로 37개 항목의 버그/보안 취약점을 발굴. 단순 코드 리뷰로는 놓칠 수 있는 다층 공격 벡터까지 발견.

2. **설계 문서의 고품질**: Design 단계에서 Before/After 코드 스펙을 명시했으므로 구현 시 편차 최소화. Match Rate 100% 달성의 주요 요인.

3. **점진적 우선순위 적용**: Critical → High → Medium 순으로 수정하면서 즉시 영향도 높은 항목부터 패치. 사용자 이슈(리버튼 오류) 조속 해결.

4. **Gap Analysis의 효율성**: gap-detector 에이전트가 resetBtn 스코프 오류를 추가로 발견. 자동화된 검증이 수동 리뷰의 맹점을 보완.

### 7.2 What Needs Improvement (Problem)

1. **초기 분석 범위 정확도**: 총 37개 항목 중 16개만 Priority 1-3에 포함, 나머지 21개는 Low/미분류. 초기 분석 시 우선순위 기준이 명확하지 않음.

2. **테스트 자동화 부재**: 각 버그 수정 후 회귀 테스트를 수동으로 진행. 특히 타이머, 메모리 누수 등은 자동화 테스트가 어려움.

3. **Edge Function 배포 프로세스 미정**: SEC-01, SEC-03 수정 후 `npx supabase functions deploy` 명령이 필요하지만 실행 환경 미확인.

4. **문서와 구현의 버전 불일치 가능성**: BUG-NEW(resetBtn)처럼 Design 문서에 포함되지 않은 버그가 발견될 수 있으므로 구현 단계 중 새로운 이슈를 캐치할 메커니즘 필요.

### 7.3 What to Try Next (Try)

1. **자동 보안 스캔 CI/CD 통합**: 매 커밋 시 security-architect 에이전트를 자동 실행하여 신규 취약점 조기 탐지.

2. **회귀 테스트 자동화 (E2E)**: Playwright/Cypress로 주요 기능(댓글 CRUD, 리버튼, 타이머 등)의 E2E 테스트 작성. 향후 수정 시 안전성 보장.

3. **설계 검증 체크리스트**: Design 단계에서 "모든 함수의 스코프 검토", "리소스 정리(revoke, clearTimeout) 확인" 등의 체크리스트 추가.

4. **배포 자동화**: GitHub Actions로 Edge Function 배포를 자동화하여 함수 수정 후 수동 배포 단계 제거.
